# Registro de Servicios

> **Regla de Actualización (OBLIGATORIA):** El Agente DEBE usar sus herramientas de sistema (escritura de archivos) para registrar el servicio en la tabla correspondiente al crearlo. Solo si el entorno no los soporta, usa el bloque `<memory_update>` para el humano.

## 2. Facades & Feature-Specific State
Servicios que median entre la UI (`features/`) y las APIs de datos (`SupabaseService`/`HttpClient`). Manejan el estado del dominio (`toSignal()`).

| Facade | Manejo de Dominio | Ubicación (File Path) | Dependencias | Estado |
|--------|-------------------|-----------------------|--------------|--------|
| `AuthFacade` | Autenticación con Supabase; expone `currentUser`, `isAuthenticated`, `login()`, `logout()`, `whenReady` | `core/facades/auth.facade.ts` | SupabaseService, Router | ✅ Estable |
| `DashboardFacade` | Estado global del Dashboard Admin; expone `loading`, `data`, `error`, `loadDashboardData()`. Mock data temporal. | `core/facades/dashboard.facade.ts` | HttpClient | ✅ Estable (mock) |
| `EnrollmentFacade` | Orquestador del wizard de matrícula (6 pasos). Draft progresivo con `expires_at` +24h. **Step 1:** `findUserByRut()`, `savePersonalData()` → upsert user+student, crear enrollment draft. **Step 2:** `loadInstructors()` (filtra `vehicle_assignments.end_date IS NULL` → solo vehículo activo), `loadScheduleGrid()` (vista `v_class_b_schedule_availability`; slot IDs son TIMESTAMPTZ directo de BD, hora en `America/Santiago`), `loadPromotions()`, `saveAssignment()` → persiste TIMESTAMPTZ sin reconstrucción. `_requiredSlotCount` computed privado (derivado de `paymentMode` + `practical_hours` del curso) valida `isStep2Complete()`. **Realtime:** `loadScheduleGrid()` suscribe a canal Realtime filtrado por `instructor_id` en `class_b_sessions`; cambios disparan re-query debounced (300ms) que auto-deselecciona slots ocupados. `reset()` limpia canal. **Step 5:** `generateContract()` (Edge Function), `uploadSignedContract()` → storage + `digital_contracts` + `contract_accepted=true` + `goToStep(6)`. `markContractSigned({ signerName, signatureHash, signedAt })` → firma digital sin archivo físico. **Step 6:** `confirmEnrollment()` → genera número YYYY-NNNN, status → active. **Wizard:** signals privados (readonly), computed `canAdvance`, `studentSummary`, `sidebarSummary`. `courseOptions` computed public (`CourseOption[]`). `reset()`, `goToStep()`, `goBack()`. | `core/facades/enrollment.facade.ts` | SupabaseService | ✅ Estable |
| `EnrollmentDocumentsFacade` | Step 3 del wizard: documentos + storage. `uploadCarnetPhoto()` (data URL → blob → storage), `uploadDocument()` / `removeDocument()` (profesional: HVC, cédula, licencia; menores: autorización notarial). `validateHvcDate()` (RF-082.3, max 30 días). `loadDocuments()` (re-edición drafts). `markDocsComplete()` → actualiza `enrollments.docs_complete`. `allRequiredUploaded()` verifica completitud. Computed `uploadedCount`. `reset()`, `getRequirements()`. | `core/facades/enrollment-documents.facade.ts` | SupabaseService | ✅ Estable |
| `EnrollmentPaymentFacade` | Step 4 del wizard: pagos + descuentos. `computePricing()` (base, deposit 50%, amountDue). `loadAvailableDiscounts()` → tabla `discounts` filtrada por curso y vigencia. `applyPredefinedDiscount()` (porcentaje o fijo). `recordPayment()` → insert en `payments`, insert en `discount_applications`, update `enrollments` (payment\_status, total\_paid, pending\_balance, discount). Computed `totalToPay`, `canConfirmPayment`. `setPaymentMethod()`, `setDiscount()`, `reset()`. | `core/facades/enrollment-payment.facade.ts` | SupabaseService | ✅ Estable |
| `PublicEnrollmentFacade` | Matrícula pública online (sin auth). Separada de `EnrollmentFacade` porque: no depende de `AuthFacade`, escribe via Edge Functions, flujo de pasos diferente. Signals: `branches`, `selectedBranch`, `flowType`, `currentStep`, `steps`, `personalData`, `instructors`, `scheduleGrid`, `selectedSlotIds`, `paymentMode`, `result`, `isSubmitting`, `isLoading`, `error`. Métodos: `loadBranches()`, `selectBranch()`, `selectFlowType()`, `confirmBranchSelection()`, `savePersonalData()`, `setPaymentMode()`, `confirmPaymentMode()`, `loadInstructors()` (Edge Fn), `loadScheduleGrid()` (Edge Fn), `toggleSlot()`, `confirmSchedule()`, `submitClaseBEnrollment()` (Edge Fn), `submitPreInscription()` (Edge Fn), `goBack()`, `reset()`. **Polling:** `loadScheduleGrid()` inicia polling cada 15s via Edge Function (Realtime no disponible para anón); auto-deselecciona slots ocupados. `reset()` detiene polling. | `core/facades/public-enrollment.facade.ts` | SupabaseService | ✅ Estable |
